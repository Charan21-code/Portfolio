*   **Tag:** [ Hackathon Winner // Climate Tech & GIS ]
*   **The Problem:** Urban heat islands make daytime navigation hazardous. Standard mapping engines optimize purely for time, ignoring thermal stress.
*   **The Solution:** An intelligent routing engine that routes users along the coolest urban paths using satellite-fetched tree canopy data[cite: 1]. It features a municipal dashboard recommending optimal tree-planting zones to combat urban heat[cite: 1].
*   **Tech Stack:** Python, FastAPI, Next.js, QGIS/GeoPandas.
*   **Impact:** 2nd Place at the Unnati Hackathon (Sustainability Domain)[cite: 1].

### System 02: AI-Driven SOAR Platform
*   **Tag:** [ Hackathon Winner // Cybersecurity ]
*   **The Problem:** Network operators are overwhelmed by false positives and slow reaction times during cyber attacks.
*   **The Solution:** An end-to-end ML detection pipeline using XGBoost on network flow features[cite: 1]. It automatically blocks high-confidence threats while routing borderline cases to an admin review dashboard[cite: 1]. It includes a self-improving decision engine using SGD incremental updates and a Contextual Bandit approach[cite: 1].
*   **Tech Stack:** Python, XGBoost, Contextual Bandits, LLaMA (Ollama), Sentence Transformers, PostgreSQL[cite: 1].
*   **Impact:** 1st Place at the Cybersecurity & Intelligent Automation Hackathon among all competing teams[cite: 1].

### System 03: Offline Competitive Coding Engine (GPrime)
*   **Tag:** [ Infrastructure // Backend Scale ]
*   **The Problem:** Tech clubs lack reliable platforms for coding contests, relying on third-party tools that fail without internet access.
*   **The System:** A fully offline, locally-hosted examination platform deployed on campus LAN with Docker containers for reproducible, conflict-free setup[cite: 1]. It features a Redis-backed submission queue that buffers 100+ simultaneous code submissions, delivering instant acknowledgment while the judge processes entries[cite: 1].
*   **Tech Stack:** Socket.io, PostgreSQL, Redis, Docker, Ubuntu Server[cite: 1].

### System 04: CV Accident Detection & Emergency Pipeline
*   **Tag:** [ Computer Vision // Public Safety ]
*   **The Problem:** Active driver-assistance systems are purely reactive, flagging danger only after a collision trajectory begins.
*   **The Solution:** A fine-tuned YOLOv8 model for real-time accident detection on live CCTV feeds[cite: 1]. Upon detection, SMTP alerts are instantly dispatched to nearest hospitals, triggering a portal for emergency dispatch[cite: 1]. It also monitors for driver distress signals and erratic lane shifts[cite: 1].
*   **Tech Stack:** Python, YOLOv8, OpenCV, Mediapipe[cite: 1].

### System 05: Apex Productivity App

*   **The System:** A production-grade web app integrating the Gemini AI API for real-time habit coaching and motivation[cite: 1]. It features an RPG-style progression engine where users earn XP and level up, backed by Firebase Firestore for live cross-session data sync[cite: 1].
